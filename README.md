Analytics AI Platform
Описание проекта
Данный проект представляет собой аналитическую платформу с интеграцией AI-агента, предназначенную для обработки естественно-языковых запросов и получения бизнес-аналитики на основе данных интернет-магазина.
Основная идея системы — пользователь формулирует запрос на естественном языке (например: «выведи продажи iPhone за последний месяц»), после чего AI-агент интерпретирует запрос, вызывает соответствующие инструменты (tools) и возвращает структурированные аналитические данные.
В качестве AI-агента используется GigaChat, который выступает в роли оркестратора между пользовательским запросом и аналитическим слоем системы.

Основной функционал


Анализ продаж в различных разрезах (по товарам, брендам, типам продукции, периодам)


Обработка естественно-языковых запросов через AI-агента


Динамическое выполнение аналитических операций через tools-методы


Построение отчетов по заказам и продажам


Гибкая система расширения аналитических функций через репозиторий аналитики



Основные сущности
Система построена вокруг следующих ключевых доменных моделей:


Product — товар в системе


Order — заказ пользователя


OrderItem — позиция товара в заказе


Brand — бренд товара


ProductType — тип товара


Эти сущности являются основой для построения всех аналитических запросов.

Архитектура системы
1. AI Agent Layer (GigaChat)
AI-агент принимает пользовательский запрос и выполняет следующие шаги:


Анализирует текст запроса


Определяет необходимую аналитическую операцию


Выбирает соответствующий tool


Вызывает метод из AnalyticsRepository


Формирует итоговый ответ



2. Tools Layer
Tools — это набор функций, которые доступны AI-агенту. Каждый tool:


соответствует конкретной аналитической операции


инкапсулирует бизнес-логику получения данных


вызывает методы из AnalyticsRepository


Примеры tools:


get_sales_by_period


get_top_selling_products


get_sales_by_brand


get_sales_by_product_type



3. Analytics Repository Layer
AnalyticsRepository — слой доступа к данным, который:


выполняет SQL/ORM-запросы


агрегирует данные


предоставляет унифицированные методы для аналитики


Именно этот слой является источником данных для всех tools.

Пример работы системы
Пользовательский запрос:
выведи продажи iPhone за последний месяц
Обработка системой:


GigaChat анализирует запрос


Определяет:


продукт: iPhone


период: последний месяц


тип операции: анализ продаж




Выбирает tool: get_sales_by_product_and_period


Вызывает метод репозитория


Получает агрегированные данные


Формирует ответ пользователю



Преимущества архитектуры


Разделение ответственности между слоями


Легкое расширение аналитических функций


Возможность добавления новых tools без изменения AI-логики


Гибкая интеграция LLM (GigaChat)


Централизованная бизнес-логика в репозитории

SQLAlchemy / ORM

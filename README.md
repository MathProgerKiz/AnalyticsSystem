# Analytics AI Platform

## Описание проекта

Analytics AI Platform — аналитическая backend-платформа с интеграцией AI-агента, предназначенная для обработки естественно-языковых запросов и получения бизнес-аналитики на основе данных интернет-магазина.

Основная идея системы заключается в том, что пользователь формулирует запрос на естественном языке, например:

> «выведи продажи iPhone за последний месяц»

После этого AI-агент интерпретирует запрос, определяет необходимые параметры, выбирает соответствующие аналитические инструменты (tools) и возвращает структурированный результат.

В качестве AI-агента используется GigaChat, который выступает оркестратором между пользовательским запросом и аналитическим слоем системы.

---

## Основной функционал

- анализ продаж в различных разрезах: товары, бренды, типы продукции, периоды;
- обработка естественно-языковых запросов через AI-агента;
- динамическое выполнение аналитических операций через tools;
- построение отчетов по заказам и продажам;
- расширяемая система аналитики через репозиторий данных;
- гибкая архитектура для добавления новых аналитических сценариев без изменения AI-логики.

---

## Основные сущности

Система построена вокруг следующих доменных моделей:

- **Product** — товар в системе;
- **Order** — заказ пользователя;
- **OrderItem** — позиция товара в заказе;
- **Brand** — бренд товара;
- **ProductType** — тип товара.

Эти сущности являются основой для всех аналитических запросов и агрегаций.

## Архитектура системы

### AI Agent Layer (GigaChat)

AI-агент является центральным звеном системы и выполняет следующие задачи:

- анализирует пользовательский запрос на естественном языке;
- извлекает ключевые сущности (товар, период, бренд и т.д.);
- определяет тип аналитической операции;
- выбирает соответствующий `tool`;
- инициирует вызов метода из `AnalyticsRepository`;
- формирует финальный ответ пользователю.

---

### Tools Layer

`Tools` — это промежуточный слой функций, доступных AI-агенту.

Каждый `tool`:

- соответствует конкретной аналитической операции;
- инкапсулирует бизнес-логику обработки данных;
- вызывает методы `AnalyticsRepository`;
- служит мостом между AI и доменной логикой.

Примеры tools:

- `get_sales_by_period`
- `get_top_selling_products`
- `get_sales_by_brand`
- `get_sales_by_product_type`

---

### Analytics Repository Layer

`AnalyticsRepository` — слой доступа к данным и бизнес-аналитики.

Он отвечает за:

- выполнение SQL/ORM-запросов;
- агрегацию и обработку данных;
- предоставление унифицированного интерфейса для аналитических операций;
- централизованную бизнес-логику получения метрик.

Именно этот слой является источником данных для всех `tools`.

---

## Пример работы системы
Пользователь отправляет запрос: выведи продажи iPhone за последний месяц

Далее система выполняет следующие шаги:

1. AI-агент (GigaChat) анализирует запрос;
2. выделяет параметры:
   - продукт: iPhone
   - период: последний месяц
   - операция: анализ продаж
3. выбирает `tool`: `get_sales_by_product_and_period`;
4. вызывает метод из `AnalyticsRepository`;
5. получает агрегированные данные;
6. формирует и возвращает результат пользователю.

## Преимущества архитектуры

- четкое разделение ответственности между слоями;
- расширяемость аналитических функций без изменения AI-логики;
- единая точка бизнес-логики в репозитории;
- гибкая интеграция LLM (GigaChat);
- простое добавление новых аналитических сценариев;
- масштабируемая архитектура для роста проекта.
